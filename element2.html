What is Typography?

Understanding why text appearance matters

Typography refers to the style and appearance of text on a web page. It involves choosing the right font, size, colour, and spacing to make text readable and visually appealing. Good typography makes content easy to read and helps the user experience.

🔑 Typography Properties in CSS
  • font-family — the style/shape of letters
  • font-size — how big or small the text is
  • font-style — normal or italic
  • color — the colour of the text
  • text-align — left, right, center, or justify
  • letter-spacing — space between each letter
  • line-height — space between lines of text
🔤

Choosing the shape and style of your letters

The font-family property changes the design of the letters. Think of it like choosing a pen style — some letters look like printed books, some look like newspaper text, and some look like a typewriter.

📄 Example — font-family in CSS
body {
  font-family: Arial, sans-serif;  /* Arial first, then any sans-serif as backup */
}

h1 {
  font-family: 'Times New Roman', serif;
}

.code-text {
  font-family: 'Courier New', monospace;  /* typewriter style */
}
💡 Always Write a Backup Font

Always list a second font (like sans-serif) after your main font. If the first font is not available on the user's computer, the browser uses the backup font.

Making text bigger or smaller

The font-size property controls how large or small text appears. You can use pixels (px), percentages (%), or relative units (em, rem).

📄 Example — font-size in CSS
p  { font-size: 16px; }      /* 16 pixels — normal reading size */
h1 { font-size: 28px; }      /* large heading */

📄 Example — All typography properties together
p {
  font-style: italic;          /* slanted text */
  color: blue;                  /* blue text */
  text-align: center;          /* centred */
  letter-spacing: 2px;         /* 2px gap between letters */
  line-height: 1.5;            /* 1.5× normal line gap */
}

Full Working Example — Mount Kilimanjaro Page

This example uses internal CSS inside the <style> tag
📌 Two ways to add CSS to HTML
  • Internal CSS: CSS written inside <style> tags in the HTML file head.
  • External CSS: CSS written in a separate .css file and linked with <link>. This is the best practice.
📄 HTML file with internal CSS (internal CSS = inside <style> tags)
<!DOCTYPE html>
<html>
  <head>
    <title>Mount Kilimanjaro</title>
    <style>
      /* This is internal CSS */
      body {
        font-family: Arial, sans-serif;
        background-color: #F5F5F5;
      }
      h1 {
        text-align: center;
        color: white;
        background-color: #333;   /* short hex for dark grey */
        padding: 20px;
      }
      p {
        font-size: 16px;
        line-height: 1.6;
        color: #333;
      }
    </style>
  </head>
  <body>
    <h1>Mount Kilimanjaro</h1>
    <h2>The Peak of Africa</h2>
    <p>Mount Kilimanjaro is a dormant volcano in Tanzania.
    It is Africa's highest peak, standing at 5,895 metres.</p>
  </body>
</html>

External CSS — The Best Practice

Keeping your CSS in a separate file

External CSS means saving all your CSS in a separate file called styles.css. You then link it to your HTML using a <link> tag. This is the best and most professional way to write CSS.

📄 styles.css — the external CSS file
/* Basic page styling */
body {
  font-family: Arial, sans-serif;
  margin: 0;
  padding: 0;
}

.header {
  background-color: #333;
  color: white;
  padding: 20px;
  text-align: center;
}

.content {
  max-width: 1200px;
  margin: 20px auto;
  padding: 20px;
  background-color: #F4F4F4;
}

/* Responsive: on small screens, add more padding */
@media (max-width: 768px) {
  .header  { padding: 10px; }
  .content { padding: 10px; }
}
💡 Link the CSS file inside <head>
<link rel="stylesheet" href="styles.css">

Both files must be in the same folder for this to work.
